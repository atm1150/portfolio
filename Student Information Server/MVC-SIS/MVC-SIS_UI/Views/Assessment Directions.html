    Assessment: Student Information System

    • Due No Due Date
    • Points 100
    • Submitting a website url

    Overview

    You have been hired at the SG College to help complete their SIS system. It has been under development but has an outstanding list of tasks that need to be completed. Your job is to familiarize yourself with the application as it exists currently and evaluate, plan, and implement the additional features specified for the initial release of the application.

    Getting Started

    Download the following file to your computer. Unzip the folder to your repository and use the unzipped files as a starting point for this assessment.

    • MVC-SIS.zip

    Contact your instructor if you are not sure where to put the files in your repository.

    Before starting to code, take time to examine the existing files and code to make sure you understand the work that has already been completed. You are encouraged to create flowcharts, pseudocode, and UML diagrams if they are useful to you.

    Requirements

    Your completed project must demonstrate the following skills:

    • Use the ASP.Net MVC framework to separate code into appropriate tiers.
    • Validate user input on both the client side and the server side.
    • Include all specifications outlined below.

    After completing all steps of the assignment, review your work to make sure you have met all expectations outlined here (including input validation) before submitting the work to your instructor.

    Specifications

    Admin Screens

    The following is a list of screens in the admin area of the web application.

    Majors Administration

    Administration of college majors is completed in Views/Admin You should review this code to understand the workflow of administration pages.

    States Administration

    You must complete the administration screens for managing the states that we offer services in. We are currently in Ohio, Kentucky, and Minnesota, but as we expand, we must be able to incorporate and manage additional states.

    The repository for states is complete. The remaining tasks are detailed below.

    1. Locate the _Layout.cshtml page in Views/Shared and create a link in the main menu under the admin section that will load the State list page.
    2. Create a method in the Admin Controller for States. This method should return a view to lists all the existing states:
      • The return type for this method should be ActionResult.
      • This method should be created as a HttpGet. (This is the default so an annotation denoting this is optional.)
      • The method should get a list of existing states from the repository and pass it as a parameter to the view returned by the method.
    3. Create the states list view.
      • Right-click the controller method name and choose Add View.
      • Select Empty (without model).
      • Leave the default view name in the dialog. It should match the name of the method you are creating it from.
      • Leave the Use a layout page checkbox as checked.
      • Click Add.

        NOTE

        Once the view is created, it will appear in the Views/Admin folder under your MVC project in the Solution Explorer. By convention, MVC looks for your view in a folder under Views that matches the name of the controller from which it was created.

      • Add the correct declaration at the top of the view to indicate the type of model that will be passed to the view (List<State>).
      • In the view, create some HTML that contains a table to display all states currently stored in our data repository.
        • There should be a column header for State Abbreviation and one for State Name.
        • Use Razor syntax in the table to create a foreach loop that will display each of the states.
      • Add a button/link to the States list view that will call the AddState method in the Admin controller.
    4. Create the AddState ViewModel.
      • Right-click the Model Folder in the UI Project and Add a new Class.
      • Name this class AddStateVM.
      • Add a single property to this, a State object called currentState.
      • We will use IValidatable Object for validations, so have it implement the IValidatable Object Interface. (See the AddEditMajorVM for an example.)
      • We want to make sure that the StateName and StaveAbbreviation are not blank or null, so add to the Validate method to make sure this cannot happen. (See the AddEditMajorVM for an example.)
    5. Set up the admin controller for Adding State.
      • Create a GET method in the Admin controller for adding states (AddState). It should return a view with an empty state model(AddStateVM) as a parameter.
      • Create the POST method in the Admin controller for adding states (AddState).
        • The method should redirect the user to the States list view upon successful completion of the Add operation.
        • If not correct, it should redirect the user back to the AddState View showing the ValidationSummary at the top.
    6. Create the AddState View.
      • Right-click the controller method name and choose Add View.
      • Select Empty (without model).
      • Leave the default view name in the dialog. It should match the name of the method you are creating it from.
      • Leave the Use a layout page checkbox as checked.
      • Click Add.
      • Add the correct declaration at the top of the view to indicate the type of model that will be passed to the view (AddStateVM).
      • In the view, create a form that will collect the StateAbbreviation and StateName from the user and POST it back to the AddState HttpPost method in the Admin controller.
      • Add a link to the States list view table (one link per row of data) that will call the EditState method in the Admin controller, passing the StateAbbreviation of the specific row to the controller to identify which one to edit.
    7. Set up the admin controller for deleting states.
      • Add a GET method in the Admin controller for DeleteState. The method should take a string parameter for StateAbbreviation.
      • Create the POST method in the Admin controller for deleting states (DeleteState). The method should redirect the user to the States list view upon the successful completion of the Delete operation.
    8. Create the DeleteState View
      • The view creation processes should be similar to those specified in previous steps.
      • Once the view is created, add a form that will contain a hidden field for StateAbbreviation that will post the data back to the server.
      • The view should warn the user of the impending deletion and allow them to proceed with the delete or cancel.
      • If the user chooses to cancel, redirect them to the States list view.
      • If the user chooses to proceed with the delete, post the form to the DeleteState Post method in the Admin controller.

    Courses Administration

    There is a need for administration screens to support Listing, Adding, Editing, and Deleting courses offered. Implement these screens in a similar fashion to the existing admin screens.

    Student Information Screens

    The Student Information section of the site is partially complete. Your task is to finish the section:

    Student List View

    The Student List view is complete with the exception of adding links to each row of data in the table to allow for editing and deleting students.

    Student Add View

    The Student Add view is complete but should be manually tested to ensure proper functionality.

    • You must add validation to the StudentAddVM to ensure a user can not add invalid/blank data. (This DOES NOT include fields for the student address.)

    Student Edit View

    Add a for editing student data. This will require additions to the Student controller as well.

    The Student Edit view should contain a form for editing the personal details as created in the Student Add view.

    • There should be an additional form alongside the personal details that will allow for saving the address of the student. You will need to create a clone of the StudentAddVM and call it StudentEditVM, as it will have its own validations. (This DOES include fields for the student address.)

    Student Delete View

    Add a view for confirming the user wishes to delete the specified student.

    Submitting your Assessment

    When you are satisfied that all of the objectives are completed take the following actions:

    1. Commit/Push your code to GitHub and complete a Pull Request.
    2. Submit the link to your GitHub Link on Canvas.
    3. A staff member will schedule a time to review your code with you. Be prepared to answer questions about your code and thought processes.
